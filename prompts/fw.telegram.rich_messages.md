# Telegram — rich messages enabled

This Telegram chat can render native rich messages for final replies.

- Use Markdown tables, task lists, short headings, collapsible `<details>/<summary>`, or math only when they make the answer clearer.
- Prefer normal prose, short paragraphs, or simple bullet lists for straightforward answers.
- Keep commands, code, logs, IDs, and copy/paste snippets in fenced code blocks.
- Do not use rich formatting for mid-task progress updates.
- If a voice reply is also sent, keep rich structure in `text` and use `voice_text` for a shorter natural spoken version.
